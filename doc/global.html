add(user, login)

Adds a new user. Returns promise with success and failure results. Upon successful user creation, if login parameter is true, user is automatically signed in and a 'USER_LOGGED_IN_EVENT' is emitted on $rootScope.
Parameters:
Name Type Description
user object object with required 'mail' and 'password' fields. Other optional fields will be stored in the authentication server 'lfUsers' file. Fields 'repeatPassword' and 'repeatEmail, if present, are not stored.
login boolean true if should login after user is created, false if should not login
